# Implementation Plan: Header and Footer with Shared Palette and Breadcrumbs

**Branch**: `013-header-footer` | **Date**: 2026-02-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/013-header-footer/spec.md`

## Summary

Add a header and footer to Ambassy using the same palette and breadcrumb approach as eventuate, foretoken, and pr-by-pt. The header includes a breadcrumb nav (johnsy.com → parkrun utilities → Ambassy), app title, subtitle, skip link, and action buttons; the footer is a single paragraph with version, author, and license. Use the shared parkrun colour palette (CSS variables in `:root`), Atkinson Hyperlegible, full-width breakout for embedded layouts, and matching accessibility (skip link, ARIA, keyboard).

**Technical Approach**: Implement in existing `public/index.html` and `public/style.css`. Add parkrun `:root` variables, skip link, breadcrumb `<nav>`, header/footer structure and styles aligned with eventuate/foretoken/pr-by-pt (including full-width breakout and breadcrumb pill styling). No new backend or API; presentational only.
